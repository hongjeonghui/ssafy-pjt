## 🗂️ Project Structure
```
📦 data-web-project
 ┣ 📁 data/                  ← 데이터 (원본, 전처리, 피처엔지니어링)
 ┣ 📁 web/                   ← 웹 구현 (백엔드 + 프론트엔드 통합)
 ┣ 📁 notebooks/             ← 분석 및 실험용 주피터 노트북
 ┣ 📁 datasets/              ← 데이터셋 (대용량 파일은 .gitignore 처리)
 ┣ 📁 results/               ← 모델 결과, 시각화, 보고서 등
 ┣ 📁 docs/                  ← 회의록, 명세 등
 ┣ 📄 requirements.txt       ← 패키지 의존성 목록
 ┣ 📄 .gitignore             ← 업로드 제외 파일 설정
 ┗ 📄 README.md              ← 프로젝트 개요 문서
```

## 🤝 Collaboration Rules

### 🧩 Branch Strategy
| 브랜치명 | 설명 |
|-----------|------|
| **main** | 최종 배포용 브랜치 (항상 안정적인 코드 유지) |
| **develop** | 통합 개발 브랜치 (기능 병합 및 테스트) |
| **data** | 데이터 처리 및 관리 |
| **feature** | 기능 단위 개발 브랜치 (ex. `feature/data-cleaning`, `feature/web-login`) |
| **docs** | 회의록, 발표 자료 등 업로드 |


### 💬 Commit Convention

| 유형 | 예시 | 설명 |
|------|------|------|
| **Add:** | `Add: 데이터 전처리 함수 추가` | 새로운 기능 추가 |
| **Fix:** | `Fix: 전처리 오류 수정` | 버그 수정 |
| **Update:** | `Update: 시각화 코드 개선` | 기능 개선 및 리팩토링 |
| **Remove:** | `Remove: 불필요한 파일 삭제` | 코드/파일 제거 |
| **Docs:** | `Docs: 회의록 업데이트` | 문서 수정 및 추가 |

> 📌 **Tip:**  
> 커밋 메시지는 **대문자로 시작**, `:` 뒤에는 **공백 한 칸**을 둡니다.
